## 第一部分：变量与基本类型 (10分钟)

**C++ 类比**：
*   TS 的 `const` 类似于 C++ 的 `const`（不可变引用）。
*   TS 的 `let` 类似于普通变量声明。
*   TS 有极强的**类型推导**，类似 C++ 的 `auto`，很多时候你不需要显式写出类型。

把下面代码复制到左侧编辑器：

```typescript
// 1. 显式类型声明 (类似 int count = 10;)
let batteryLevel: number = 80;
let agvName: string = "AGV-01";
let isRunning: boolean = true;

// 2. 类型推导 (类似 auto name = "AGV-02";)
let targetPoint = "Point-A"; // TS 自动推导出 targetPoint 是 string
// targetPoint = 100; // ❌ 尝试取消注释这行，你会看到熟悉的编译报错：不能将 number 分配给 string

// 3. 数组 (类似 std::vector<string>)
let paths: string[] = ["Point-A", "Point-B", "Point-C"];
// 或者泛型写法：let paths: Array<string> = ...

// 4. 只读常量 (类似 const float PI = 3.14;)
const maxSpeed: number = 1.5; 
// maxSpeed = 2.0; // ❌ 报错：Cannot assign to 'maxSpeed' because it is a constant.

console.log(`车辆 ${agvName} 电量: ${batteryLevel}%`); // 点击顶部 Run 看看输出
```

## 第二部分：Interface (接口) —— 你的核心武器 (20分钟)

**C++ 类比**：`interface` 在 TS 中最常用的功能，相当于 C++ 中的 `struct`（定义数据聚合）或**纯虚类**（定义行为契约）。在前端 SPA 开发中，**90% 的情况你用它来描述与 BFF 交互的 JSON 数据长什么样**。

清空编辑器，粘贴以下代码：

```typescript
// 定义一个数据结构 (类似 C++ 的 struct)
interface Point {
    x: number;
    y: number;
    name?: string;       // 加 '?' 表示可选字段 (类似 std::optional<string>)
    readonly id: number; // 只读字段 (初始化后不可修改)
}

// 定义一个 AGV 状态接口
interface AgvState {
    vehicleId: string;
    battery: number;
    position: Point;     // 嵌套结构
}

// 实例化一个对象 (这里类似 JSON，必须严格符合 interface 定义)
const agv1: AgvState = {
    vehicleId: "AGV-001",
    battery: 85,
    position: {
        id: 1001,
        x: 15.5,
        y: 20.0
        // name 没写，因为是可选的，完全合法
    }
};

// agv1.position.id = 1002; // ❌ 报错：id 是 readonly 的
// agv1.speed = 10;         // ❌ 报错：AgvState 中没有定义 speed 属性

console.log(agv1.vehicleId, "位于坐标", agv1.position.x, agv1.position.y);
```

> **提示**：TS 的 interface 是“鸭子类型”（Duck Typing）。只要你的长得像鸭子（属性对得上），TS 就认为你是鸭子。你不需要像 C++ 那样显式去 `class MyStruct : public IInterface`。

## 第三部分：泛型 (Generics) —— C++ 模板的亲兄弟 (15分钟)

**C++ 类比**：TS 的泛型 `<T>` 几乎和 C++ 的 `template <typename T>` 一模一样，用于编写可复用的代码，特别是**封装通用 HTTP 响应结构**时必不可少。

清空编辑器，粘贴以下代码：

```typescript
// 假设 BFF 所有的 REST 接口返回格式都是统一的结构：
// { "code": 200, "message": "success", "data": { ...具体数据... } }

// 就像 C++ 的 template <typename T> struct ApiResponse { ... }
interface ApiResponse<T> {
    code: number;
    message: string;
    data: T;
}

// 我们之前定义的实体
interface Vehicle {
    name: string;
    state: "IDLE" | "EXECUTING" | "ERROR"; // 这是联合类型（类似 Enum）
}

interface MapInfo {
    resolution: number;
    originX: number;
    originY: number;
}

// 模拟解析 BFF 接口返回的 JSON
// 注意这里的类型约束：ApiResponse<Vehicle>
const fetchVehicleResponse: ApiResponse<Vehicle> = {
    code: 200,
    message: "OK",
    data: {
        name: "AGV-01",
        state: "IDLE" // 如果你敲错成 "STOP"，这里会直接标红！这就是 TS 的强大之处
    }
};

// 换一个数据实体，复用了 ApiResponse 结构
const fetchMapResponse: ApiResponse<MapInfo> = {
    code: 200,
    message: "OK",
    data: {
        resolution: 0.05,
        originX: -10,
        originY: -10
    }
};

console.log(fetchVehicleResponse.data.name);
```

## 第四部分：实操检验挑战 (15分钟)

在实际的 SPA MVP 中，我们经常需要处理分页列表数据。现在请你在 TypeScript Playground 中完成以下任务。

**需求：**
1. 定义一个 `PageData<T>` 泛型接口，包含三个字段：
   - `totalItems`: 数字类型（总条数）
   - `currentPage`: 数字类型（当前页码）
   - `items`: 数组类型，内部元素类型为 `T`
2. 定义一个 `TransportOrder`（运输订单）接口，包含：
   - `orderId`: 字符串
   - `destPoint`: 字符串
   - `isCompleted`: 布尔值
3. 创建一个变量 `mockOrderPage`，类型定为 `PageData<TransportOrder>`，并随便填入两个订单的假数据。

**尝试自己写，感受一下 IDE 的代码补全和错误提示。卡住了请看下方参考答案：**

```typescript
// 1. 定义泛型分页接口 (相当于 template <typename T> struct)
interface PageData<T> {
    totalItems: number;
    currentPage: number;
    items: T[]; // 或者 Array<T>
}

// 2. 定义业务实体接口
interface TransportOrder {
    orderId: string;
    destPoint: string;
    isCompleted: boolean;
}

// 3. 组合并赋值
const mockOrderPage: PageData<TransportOrder> = {
    totalItems: 50,
    currentPage: 1,
    items: [
        {
            orderId: "TCS-1001",
            destPoint: "Station-A",
            isCompleted: true
        },
        {
            orderId: "TCS-1002",
            destPoint: "Station-B",
            isCompleted: false
        }
    ]
};

// 你可以尝试敲出 mockOrderPage.items[0]. 
// 看看编辑器是不是自动提示出了 orderId, destPoint 等字段？这就是 TS 最爽的地方！
```

## 总结你的 1 小时收获

1. 你不需要学习新的面向对象心智，`interface` 就是你的 `struct` 和契约。
2. `<T>` 泛型的玩法和 C++ Template 高度一致，专门用来应对嵌套和复用的 JSON 结构。
3. `?` 用来处理可选/为空的字段，有效防止前端出现各种 `undefined` 报错。

如果你理解了这些，在下一轮配置 Vite 脚手架（R1）或编写 BFF 接口调用（R2）时，你就能看懂所有数据结构是如何在组件间传递的了！
