## 第一部分：HTML —— 声明式 UI 结构 (10分钟)

**C++ 类比**：HTML 就是一棵树状的 Node 结构。标签名就是类名，属性（如 `class`, `id`）就是成员变量。你只声明结构，浏览器引擎负责解析成内存里的对象树（DOM Tree）。

把下面代码复制到 `index.html`：

```html
<!DOCTYPE html>
<html>
<head>
  <style>
    /* CSS 代码会写在这里 */
  </style>
</head>
<body>
  <!-- div 是最常用的通用容器，就像 std::any 或一个空的 Struct -->
  <div class="card">
    <h2 class="title">AGV 状态监控</h2>
    <div class="content">
      <span class="status">运行中</span>
      <span class="battery">电量 80%</span>
    </div>
  </div>
</body>
</html>
```
*保存并刷新浏览器，你会看到普通的黑白文字。*

## 第二部分：CSS 与 盒模型 (Box Model) (20分钟)

**核心心智**：浏览器里的一切元素，都是一个**矩形盒子**。
盒模型由内到外分为四层：
1. **Content (内容)**: 实际的文字或图片大小。
2. **Padding (内边距)**: 内容与边框的距离（类比相框里的留白卡纸）。
3. **Border (边框)**: 盒子的外壳（类比相框木边）。
4. **Margin (外边距)**: 盒子与其他盒子之间的距离（类比墙上两个相框的距离）。

将以下 CSS 代码覆盖到 `<style>` 标签内：

```css
/* 1. 全局重置：这是前端基操，消除浏览器的默认边距，并强制盒模型计算方式 */
* {
  box-sizing: border-box; /* 核心！让 width 包含 padding 和 border，防止盒子被撑爆 */
  margin: 0;
  padding: 0;
}

body {
  background-color: #f0f2f5;
  padding: 50px; /* 让内容离浏览器边缘远一点 */
}

/* 2. 给卡片上色并验证盒模型 */
.card {
  width: 300px;           /* 盒子宽度 */
  background-color: white;

  /* --- 盒模型三剑客 --- */
  padding: 20px;          /* 内边距：把里面的文字往里推 20px */
  border: 2px solid #ccc; /* 边框：2像素宽，实线，灰色 */
  margin-bottom: 20px;    /* 外边距：如果下面还有其他卡片，隔开 20px */
  /* ------------------ */

  border-radius: 8px;     /* 圆角（纯视觉） */
  box-shadow: 0 4px 8px rgba(0,0,0,0.1); /* 阴影（纯视觉） */
}

.title {
  color: #333;
  margin-bottom: 15px; /* 标题和下方内容隔开 */
}

.status {
  background-color: #e6f7ff;
  color: #1890ff;
  padding: 4px 8px; /* 上下4px，左右8px */
  border-radius: 4px;
}
```
*保存刷新浏览器，按 `F12` 打开开发者工具，点击 `Elements` 面板，选中 `<div class="card">`。在右侧找到 `Computed`（或 Layout）标签，你能直观看到一个彩色的同心矩形，这就是真实的物理盒模型。*

### `<style>` 元素

`<style>` 是html中专门用来写内部css样式的标签，作用是给页面元素设置外观(颜色，大小，位置，字体等)，是网页开发中的核心样式标签之一。

**基本用法**
1. 位置
  必须写在 `<head>`标签内部(也能写在body, 但规范写法是 head)

2. 语法
```html
<head>
  <!--这里写 style, 专门放css样式-->
  <style>
    /*这里面全部是css代码 */
    选择器 {
      属性: 值
    }
  </style>
</head>
```

3. 优先级
  比外部css文件优先级高，比行内style地

4. 注意事项
  - `<style>` 内部不能写 html, 只能写 css
  - css 注释是 `/* */` 不是 `<!---->`
  - 多个 `<style>` 标签可以共存，样式会合并
  - 在 vue, react 等框架中, `<style>` 是标配写法
---

### css 选择器全解

**基础语法格式**
```css
选择器 {
  样式属性: 值;
  样式属性: 值;
}
```
示例:
```css
p {
  color: red;
  font-size: 16px;
}
```

**常用选择器分配**
1. 标签选择器(元素选择器)
直接写html标签名，选中页面所有同名标签。
```css
/*所有p标签 */
p {

}

/*所有 div 标签 */
div {

}

/*所有 h1标签 */
h1 {

}
```

2. class 类选择器
语法: .类名，最常用
```html
<div clss="box"></div>
```
```css
.box {

}
```

3. id 选择器
语法: #id, 唯一标识
```html
<div id="app"></div>
```
```css
#app {}
```

4. 通配符选择器
* 选中所有页面元素
```css
* {
  margin: 0;
  padding: 0;
}
```

**层级关系选择器**
1. 后代选择器 `空格`
所有后代(子，孙全部选中)
```css
#app div{}
```

2. 子代选择器 `>`
只选直接子元素
```css
#app > div {}
```

3. 相邻兄弟选择器 `+`
紧挨着的下一个同级元素
```css
p + div {}
```

4. 通用兄弟选择器 `~`
后面所有同级兄弟
```css
p ~ div {}
```

**极简速记表**
| 符号    | 含义      |
|---------|-----------|
| `div`   | 标签选择器 |
| `.name` | 类选择器  |
| `#name` | id选择器  |
| `*`     | 全部      |
| ` `空格 | 所有后代  |
| `>`     | 直接子代  |
| `,`     | 多选合并  |

## 第三部分：Flex 布局 (20分钟)

这是前端最重要的发明。以前居中一个元素需要算各种像素，现在靠 Flex。
**C++ 类比**：Flex 是一个自动计算一维坐标（X轴或Y轴）的布局引擎。你指定一个父容器为 `display: flex`，它的所有直接子元素就会变成弹性项目。

Flex 只有两个最核心的概念：
1. **主轴 (Main Axis)**：默认是水平从左到右 (`flex-direction: row`)。用 `justify-content` 控制。
2. **交叉轴 (Cross Axis)**：默认是垂直从上到下。用 `align-items` 控制。

继续在 `<style>` 中追加代码：

```css
/* 3. 给 .content 加上 Flex 魔法 */
.content {
  display: flex; /* 激活 Flex 布局，子元素默认排成一行 */

  /* 主轴（水平）对齐方式：
     flex-start (靠左), center (居中), flex-end (靠右)
     space-between (两端对齐，中间留空) -> 最常用！
  */
  justify-content: space-between;

  /* 交叉轴（垂直）对齐方式：
     flex-start (靠上), center (垂直居中), stretch (拉伸)
  */
  align-items: center;

  border-top: 1px dashed #eee; /* 加条虚线 */
  padding-top: 15px; /* 虚线离内容的距离 */
}
```
*保存刷新，你会发现“运行中”和“电量 80%”自动分别贴到了卡片的左侧和右侧，并且垂直居中对齐了。你不需要计算它们距离左边多少像素。*

## 总结你的 1 小时收获

1. 想要元素之间有距离：用 `margin`。
2. 想要边框里面有留白：用 `padding`。
3. 想要把几个按钮排成一排并对齐：给它们的父节点加 `display: flex; justify-content: ...; align-items: ...;`。

这就是现代 SPA 前端 90% 的布局日常，理解了它们，你就能看懂后面我们要写的 Vue 组件结构了。下一步，我们就可以正式进入 **R0 选型** 了！
